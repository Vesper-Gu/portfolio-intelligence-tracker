# UI 设计规范

最后更新：2026-05-22

## 设计方向

Portfolio Intelligence Tracker 的前端采用“彭博终端式金融研究工作台”方向。它不是营销页，也不是普通 SaaS dashboard，而是面向个人研究者的高密度桌面工具。

核心感受：

- 信息优先，装饰克制。
- 深色底、橙色焦点、红绿状态色。
- 命令驱动，支持快速跳转和检索。
- 多面板同屏联动，但首页只保留决策所需信息。
- 所有判断必须能追溯到来源和时间；内部评分不默认暴露给用户。

## 适用范围

Phase 1 前端只覆盖桌面宽屏，推荐设计宽度为 `1440px`。移动端不作为 MVP 验收目标。

Phase 1 页面优先级：

1. 总览工作台。
2. 录入资料。
3. 标的资料库。
4. 问资料库。
5. 设置 / 隐私与运维。

RAG 问答在 Phase 1 已接入最小闭环，当前以结构化 holdings、portfolio positions、holding events、source ingest raw text 和 extraction candidates 为证据源，生成确定性中文答案。

### 顶部命令栏

命令栏是核心入口，不只是搜索框。

示例命令：

- `NVDA <GO>`
- `KOL @Investor_X`
- `RAG TSLA Q1`
- `INGEST REVIEW`

Phase 1 行为：

- 输入内容只保存在本地状态。
- `Enter` 暂不执行真实检索。
- 页面上显示当前命令值，作为未来 command router 的占位。

### Ticker Strip

用于展示市场环境和常看标的：

- SPX
- NDX
- NVDA
- TSLA
- BTC
- ETH
- VIX
- DXY

颜色语义：

- 绿色：上涨、加仓、已验证。
- 红色：下跌、减仓、风险、需确认。
- 黄色 / 橙色：焦点、持有、待处理、警示。
- 灰色：中性、无记录、低优先级信息。

## 页面结构

### 总览工作台

目标：一屏内看清“当前聚合方向是什么、最近有什么变化、哪些资料需要处理、可以直接问什么”。

主要区域：

- 当前关注标的 / Portfolio Positions。
- 最近变化 / Recent Events。
- 待处理资料 / Review Queue。
- KOL × Ticker 热力图，作为辅助观察模块。
- 快速提问 / Ask Library。

不放在总览页：

- 持仓信号 Tape / Consensus。
- 静态 RAG 证据面板。
- Accepted Holdings / 已确认持仓明细表。
- Source Trace 详情块。
- 候选结果完整历史。
- 图片证据预览。
- 归档 / 恢复等复核操作。
- confidence、avg confidence、citation score 等内部评分。

聚合仓位字段：

- `Ticker`
- `Stance`
- `Holdings`
- `Sources`
- `Latest`
- `Updated`

交互：

- 点击“查看标的资料库”进入按 ticker 聚合的资料库页面。
- KOL 档案、标的矩阵、提醒规则和数据质量暂不在左侧导航展示，避免出现未实现入口。
- 已确认记录明细和 source trace 后续放到标的资料库或录入项详情页，不放在总览。
- 快速提问直接跳转到“问资料库”，并带入问题模板。

### 录入资料

目标：把 text / screenshot / link 资料解析后加入正式资料库。

主要区域：

- 待加入资料列表。
- 资料摘要 / 截图预览。
- 解析字段预览。
- 解析来源和候选历史摘要。
- 加入资料库、修改、驳回、标记为手动处理。

Phase 1 mock 行为：

- 使用本地 mock 数据展示状态。
- 按钮只改变本地状态或展示静态状态。

当前真实前端行为：

- 支持链接、文本和图片上传。
- 支持 AI 解析、候选历史、一键应用候选、人工修改、加入资料库和驳回。
- Ticker、Action、Summary 可在界面修改。
- Confidence 保留为内部兼容字段，不作为用户可见输入项或展示项。
- 图片上传的 storage object、文件大小和 reviewer note 不在普通用户界面直接展示；界面优先展示资料摘要。

### 设置 / 隐私与运维

目标：把上线前必须面对的数据权利、API key 状态、成本控制和隐私边界显式展示给用户。

主要区域：

- 账户与数据：导出资料库 JSON、删除当前资料库数据。
- API Key 与 Provider：只显示是否配置，不显示真实 key。
- 成本控制：显示 session 内 RAG、AI 解析、上传计数和限制。
- 隐私说明：说明截图存储、signed URL、模型调用和资料库边界。

### 标的资料库

目标：用户按 ticker 查看已经加入资料库的内容，而不是在总览里看底层记录。

主要区域：

- 每个 ticker 一张资料卡。
- 展示聚合方向、最新动作、已确认资料数量和来源。
- 展示最近确认事件摘要。
- 提供“问这个标的”和“查看依据”入口，跳转到问资料库。
- 已确认资料可直接打开统一的资料详情侧栏，查看原始图片或文本、摘要、状态和解析记录。

### 来源设置

目标：管理 KOL、平台、抓取方式和 parser 策略。

主要区域：

- 数据源列表。
- 当前来源详情。
- Parser 配置。
- 同步状态。

字段：

- 名称。
- 平台。
- 类型。
- 状态。
- 最近同步。
- 记录数。
- Parser 策略。

### RAG 问答

目标：让用户以 chatbot 形式基于自己存入并确认的数据连续提问。

主要区域：

- 聊天输入。
- 连续对话区。
- 最近一轮回答的依据资料列表。
- 资料库边界提示。
- 引用依据可点击打开资料详情侧栏，完成从回答到原始证据的核验路径。

展示规则：

- citation 显示来源类型、实体 ID 和摘要。
- 不展示检索 score。
- 回答不展示平均置信度。
- 后端接收最近对话上下文，用于理解“它”“这个标的”“证据呢”等追问。
- 如果问题无法命中用户数据，后续应明确提示“当前资料中未找到相关记录”，而不是泛化回答。

### 数据质量摘要

目标：监控 Phase 1 数据闭环，而不是展示空泛 AI 指标。

核心指标：

- Ticker 准确率。
- 字段完整率。
- 待确认记录数。
- 待人工处理记录数。

错误类型：

- OCR 表格错位。
- Ticker 歧义。
- 仓位缺失。
- 日期缺失。
- 期权字段缺失。

## 组件规范

### Panel

Panel 是终端工作台中的基础容器。

- 背景：接近黑色。
- 边框：低对比灰色。
- 圆角：`0` 到 `2px`，避免 SaaS 卡片感。
- 标题：橙色，小字号，全局一致。

### Table

表格是核心信息载体。

- 行高：`28px` 到 `32px`。
- 字号：`11px` 到 `12px`。
- 表头使用橙色。
- 单元格尽量短，长证据文本截断。
- 数字列右对齐或视觉上保持列宽稳定。

### Heatmap

热力图必须有坐标标签，不能只做装饰块。

- X 轴：ticker。
- Y 轴：KOL / source。
- 单元格颜色表达持仓状态。
- 提供简短图例。

颜色：

- 绿色：加仓 / 高共识。
- 黄色：持有 / 中性。
- 红色：减仓 / 风险。
- 深灰：无记录。

### Command Input

命令输入框使用橙色边框，位置固定在顶部。

未来应支持：

- command parsing。
- autocomplete。
- history。
- keyboard shortcut。

Phase 1 只实现本地输入状态。

## 字体与排版

- 首选字体：Inter，系统无 Inter 时使用 `ui-sans-serif`。
- 数字和 ticker 使用等宽后备：`ui-monospace`。
- 页面标题：`18px`。
- 面板标题：`11px` 到 `12px`。
- 表格文字：`11px`。
- KPI 数字：`24px` 到 `28px`。

不要使用 hero 级大标题。工作台内所有文本应保持紧凑。

## Phase 1 前端验收

- 应用能本地启动。
- 总览页展示聚合仓位、最近变化、待处理资料和快速提问。
- 可以在左侧导航切换总览、录入和问资料库。
- 命令栏输入可编辑。
- 默认可使用 mock API 运行，不依赖后端云凭证；配置 database repository 后可连接 Supabase dev 数据库。
- 页面在 `1440px` 宽度下无明显重叠。
- 不引入真实 secret 或真实私人数据。
